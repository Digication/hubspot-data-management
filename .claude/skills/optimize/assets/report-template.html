How the Optimization System Works

These diagrams explain the loop that produced the results on the first tab.

The 5 Steps Inside Every Cycle

Each cycle runs these five steps in order. After Measure, the loop checks if there are remaining issues and decides whether to start another cycle.

{{DIAGRAM_LOOP_FLOWCHART}}

How the Loop Decides to Continue or Stop

After each cycle, the system checks for remaining HIGH or MEDIUM severity issues. LOW issues alone don't trigger another cycle — they're nice-to-haves, not blockers.

{{DIAGRAM_CONTINUE_OR_STOP}}

Which Model Is Used at Each Step

Evaluation steps (Discover, Measure) use Opus because they set the quality ceiling. Writing steps (Analyze, Apply) use Sonnet — faster and cheaper, same quality for drafting.

{{DIAGRAM_MODEL_SELECTION}}

Why Scores Vary Between Steps

Each Discover and Measure step uses a fresh, independent agent — like two different teachers grading the same essay. Scores may vary ±0.5–1.0 points between steps. This is normal. The important number is the delta within each cycle, not the absolute score between cycles.
